# Web Dashboard

Современный веб-дашборд для управления порталами с темным интерфейсом, построенный на Django.

## Возможности

- ✅ **Аутентификация пользователей** - регистрация и вход в систему
- ✅ **Управление порталами** - добавление, редактирование и удаление веб-порталов через интуитивный интерфейс
- ✅ **Drag & Drop** - перемещение порталов для изменения порядка
- ✅ **Мониторинг доступности** - автоматическая проверка доступности порталов (ping)
- ✅ **Автоматические превью** - загрузка favicon для каждого портала
- ✅ **Статистика uptime** - отображение процента доступности каждого портала
- ✅ **Современный UI** - темная тема с градиентами и эффектами glassmorphism
- ✅ **Responsive дизайн** - адаптивный интерфейс для всех устройств

## Установка и запуск

### 1. Активация виртуального окружения

```bash
source venv/bin/activate
```

### 2. Установка зависимостей (если еще не установлены)

```bash
pip install -r requirements.txt
```

### 3. Применение миграций (уже выполнено)

```bash
python manage.py migrate
```

### 4. Создание суперпользователя

```bash
python manage.py createsuperuser
```

Следуйте инструкциям для создания учетной записи администратора.

### 5. Запуск сервера разработки

```bash
python manage.py runserver
```

Приложение будет доступно по адресу: http://127.0.0.1:8000/

## Использование

### Вход в систему

1. Перейдите на http://127.0.0.1:8000/
2. Введите имя пользователя и пароль
3. Или создайте новый аккаунт через регистрацию

### Добавление портала

1. Нажмите кнопку **"+ Добавить портал"**
2. Заполните форму:
   - **Название** - отображаемое имя портала
   - **URL** - полный адрес портала (например, https://google.com)
   - **Описание** (необязательно) - краткое описание портала
3. Нажмите **"Добавить"**

Система автоматически загрузит favicon и проверит доступность портала.

### Редактирование портала

1. Нажмите иконку **карандаша** на карточке портала
2. Измените данные в форме
3. Нажмите **"Сохранить"**

### Удаление портала

1. Нажмите иконку **корзины** на карточке портала
2. Подтвердите удаление

### Перемещение порталов

1. Наведите курсор на карточку портала
2. Зажмите и перетащите за иконку **⋮⋮** (drag handle)
3. Порядок сохранится автоматически

## Мониторинг доступности

### Автоматическая проверка

Запустите команду для мониторинга всех порталов:

```bash
python manage.py monitor_portals
```

### Настройка периодической проверки (cron)

Добавьте в crontab для проверки каждые 15 минут:

```bash
crontab -e
```

Добавьте строку:

```
*/15 * * * * cd /Users/gegaremant/DEV/web_dashbord && ./venv/bin/python manage.py monitor_portals
```

## Технологии

**Backend:**
- Django 5.0
- Python 3.x
- SQLite (можно заменить на PostgreSQL)

**Frontend:**
- Vanilla JavaScript
- CSS3 (переменные, градиенты, backdrop-filter)
- Sortable.js (для drag-and-drop)

**Особенности дизайна:**
- Темная тема с градиентами
- Glassmorphism эффекты
- Плавные анимации и transitions
- Responsive grid layout

## API Endpoints

- `GET /dashboard/` - Главная страница дашборда
- `POST /dashboard/portal/create/` - Создание портала
- `POST /dashboard/portal/<id>/update/` - Обновление портала
- `POST /dashboard/portal/<id>/delete/` - Удаление портала
- `POST /dashboard/portal/reorder/` - Изменение порядка
- `GET /dashboard/portal/<id>/availability/` - Статистика доступности
- `POST /dashboard/portal/<id>/check/` - Проверить доступность сейчас

## Разработка

### Активация режима отладки

В `web_dashboard/settings.py`:
```python
DEBUG = True
```

### Создание миграций после изменения моделей

```bash
python manage.py makemigrations
python manage.py migrate
```

## Производство

Для развертывания в production:

1. Установите `DEBUG = False` в settings.py
2. Настройте `ALLOWED_HOSTS`
3. Используйте PostgreSQL вместо SQLite
4. Настройте nginx + gunicorn
5. Соберите статические файлы: `python manage.py collectstatic`
6. Настройте SSL сертификаты
